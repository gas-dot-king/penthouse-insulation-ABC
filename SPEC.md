# 펜트하우스 보온재 계산기 — 상세 스펙

`penthouse_insulation_ABC_case_optimizer.html`(구조) + `styles.css` + `js/*.js`로 동작하는 계산기의 전제, 케이스 치수, 계산 로직을 정리한 문서입니다. 코드를 수정할 때 이 문서의 규칙을 먼저 확인하세요.

## 1. 개요

- 대상: 정면에서 봤을 때 ㄴ자(단차) 형태로, 긴 방향으로 이어진 펜트하우스 구조물.
- 목적: 외부 면(지붕·단차 수직면·높은 벽·낮은 벽·양끝 L자면)에 1800×600mm 보온재를 덮을 때 필요한 최소 장수를 계산.
- 형태: 값 대부분을 하드코딩한 이해용 웹페이지. 케이스별로 분리해서 관리.

## 2. 고정 전제

1. **패널 크기는 고정값** — 1800 × 600mm, 1장 면적 1.08㎡. 사용자가 수정하지 않음.
2. **바닥면은 덮지 않음** — 바닥으로 내려가는 잔여분은 잘라내는 것으로 간주. 계산 대상은 지붕, 단차 수직면, 높은 벽, 낮은 벽, 양끝 L자면뿐.
3. **높은 벽 위 기준 양방향 전개** — 높은 벽 위 모서리에서 시작해 한쪽은 높은 벽을 바닥까지 직선 하강, 반대쪽은 높은 지붕 → 단차 수직면 → 낮은 지붕 → 낮은 벽 바닥까지 접어 내려가는 것으로 계산.
   - 1800mm 방향을 각 하강 전개 방향, 600mm 방향을 길이 방향 반복 배치로 사용.
   - 정면 ㄴ자 모서리와 지붕 단차는 절단선이 아니라 접힘선으로 취급.
   - 양방향 각각의 바닥 쪽 잔여띠는 스트립 재단과 1800×600 가로 밴드를 비교해 장수가 적은 쪽을 적용.
4. **가로/세로를 한 방향으로 고정하지 않음** — 다음을 자동 비교해 장수가 적은 쪽을 선택.
   - 긴 외피 바닥 잔여띠: 600폭 절단 스트립 vs 1800×600 가로 배치 → 장수가 적은 쪽(동률이면 이음매가 적은 가로 배치 우선 표기).
5. **케이스별 선택** — 케이스 A / B / C 세 개를 전환하며 확인. (기존 "케이스 1"→A, "케이스 2"→B로 개명, C는 신규 추가)

## 5. 계산 로직

### 5.1 기본 파생값 — `base(c)`
```
lowW    = totalW − highW
stepH   = highH − lowH
drapeW  = highW + stepH + lowW        // 지붕 전개폭(단면 둘레)
lenCols600  = ceil(L ÷ 600)           // 길이 방향 600폭 열 수
lenCols1800 = ceil(L ÷ 1800)          // 길이 방향 1800폭 열 수
trim600  = 600  − (L − 600×(lenCols600−1))   // 마지막 열 절단 자투리
trim1800 = 1800 − (L − 1800×(lenCols1800−1))
roofRows = ceil(drapeW ÷ 1800)        // 지붕 전개 방향 단수
fold     = roofRows×1800 − drapeW     // 남아서 벽으로 접는 폭
```

### 5.2 긴 외피 전개 — `shellPlan(d)`
높은 벽 위 모서리를 기준으로 두 방향을 따로 계산합니다.
```
highRunW    = highH
roofLowRunW = highW + stepH + lowW + lowH

각 방향별:
fullRows   = floor(runW ÷ 1800)
mainSheets = fullRows × lenCols600
rem        = runW − fullRows×1800
```

### 5.3 바닥 잔여띠 — `bandPlan(rem, d)`
```
// 잔여 높이 처리: 두 방식 비교 후 적은 쪽 채택
stripsPerSheet   = max(1, floor(1800 ÷ rem))       // 패널 1장에서 뽑는 스트립 줄 수
stripSheets      = ceil(lenCols600 ÷ stripsPerSheet)
horizontalRows   = ceil(rem ÷ 600)
horizontalSheets = horizontalRows × lenCols1800
bandSheets        = min(stripSheets, horizontalSheets)      // 동률이면 가로 우선 표기

긴 외피 장수 = mainSheets + bandSheets
```

### 5.4 양끝 L자면
- 케이스별 하드코딩 값(§4) 사용: `endPerFace × 2면`
- 면적 하한 검산식(참고용): `ceil((highW×highH + lowW×lowH) ÷ 1,000,000 ÷ 1.08)`

### 5.5 총합 · 손실률 · 발주
```
totalSheets = shellSheets + endSheets
totalPanel  = totalSheets × 1.08                          // ㎡
totalActual = L×shellW ÷ 1,000,000 + endActual
lossRate    = (totalPanel − totalActual) ÷ totalActual × 100
발주 수량    = ceil(totalSheets × (1 + 여유율))            // 여유율 0/3/5/7/10% 선택
```

## 7. 화면 구성 (섹션 맵)

1. **케이스 선택** — A/B/C 카드(미니 정면도, 핵심 치수, 카드별 총계 미리보기)
2. **결과 요약** — 총 장수, 구역별 스택 막대, 발주 여유율, 양끝면 기준 토글, 케이스 치수 상세(접힘 기본)
3. **덮는 방법(시공 순서 5단계)** — 드레이프 → 접기 → 벽 본판 → 잔여 밴드 → 양끝면
4. **구역별 상세** — 구역 카드 4장 + 계획표(클릭 시 시각화로 이동)
5. **시각화** — **3D 덮기(기본 탭)** / 전체 브리핑 / 긴 외피 전개 / 양끝 ㄴ자면 / 막대 비교.
   - 3D 덮기: ㄴ자 건물을 캔버스 입체로 그림. 드래그 회전 · 휠 확대/축소 · 더블클릭 시점 초기화 · 시점 프리셋(비스듬히/정면 ㄴ자/위에서). 단계 버튼(구조만 → ① 높은 벽 직하강 → ② 지붕→낮은 벽 드레이프 → ③ 바닥 잔여띠 → ④ 양끝면)과 ▶ 재생 애니메이션(첫 진입 시 1회 자동 재생). 각 단계 라벨에 열×단 수량과 잔여띠 가로/스트립 비교 근거를 직접 표기. 색은 범례와 동일(파랑 본판·초록 접힘선·주황 잔여띠·빨강 자투리·보라 양끝면).
   - 양끝면·3D는 케이스별 `highSide`(A: 왼쪽, B·C: 오른쪽)에 맞춰 도면과 같은 방향으로 렌더링.
6. **비교** — `<details>`로 접힘(기본 접힌 상태). 면별 가로·세로·혼합 비교표 + A/B/C 선택 기준별 전체 비교표.
7. **검산** — `<details>`로 접힘(기본 접힌 상태). 6개 블록의 공식·중간값 전체 노출.

## 8. 알려진 제약

- 양끝 L자면 최소 재단 값은 실측 재단 시뮬레이션이 아닌 케이스별 검토값이므로, 새 케이스 추가 시 반드시 실측/재단 검토 후 `endMinPerFace`/`endPracticalPerFace`/`endNote`를 채워야 함.
- 겹침·이음 여유(시공 여유)는 수량에 포함되지 않음. 발주 여유율(0~10%)로만 보정.
- `highSide`는 시각화 미러링에만 쓰이며 수량 계산에는 영향 없음(좌우 대칭이므로).
