# 펜트하우스 보온재 계산기 — 상세 스펙

`penthouse_insulation_ABC_case_optimizer.html`(구조) + `styles.css` + `js/*.js`로 동작하는 계산기의 전제, 케이스 치수, 계산 로직을 정리한 문서입니다. 코드를 수정할 때 이 문서의 규칙을 먼저 확인하세요.

## 1. 개요

- 대상: 정면의 **직교 계단형 단면을 길이 방향으로 동일하게 압출한** 펜트하우스 구조물. 한 단, 기존 ㄴ자 두 단, 세 단 이상을 같은 데이터 구조로 표현할 수 있습니다.
- 목적: 바닥을 제외한 외부 면(시작 외벽·각 지붕·단차 수직면·반대편 외벽·양끝 단면)에 1800×600mm 보온재를 덮을 때 필요한 장수를 계산.
- 형태: 케이스의 길이와 단면 구간을 `CASES[].L`, `CASES[].shape.sections`에 선언하면 계산·2D·3D가 같은 파생 형상을 사용합니다.

### 1.1 수정 가능한 단면 모델

`shape.sections`는 **시작 외벽에서 반대편 외벽으로 진행하는 순서**로 `{width, height}`를 나열합니다. 구간 수는 1개 이상이며, 높이는 내려가거나 다시 올라가도 됩니다. 정규화 단계가 전체 폭, 최대·최소 높이, 양끝 polygon, 지붕·단차·외벽 surface, 두 전개 run과 면적을 자동으로 만듭니다.

```js
shape: {
  schemaVersion: 1,
  type: "stepped-profile",
  startSide: "left",
  startRunLabel: "높은 벽 직하강",
  profileRunLabel: "지붕→낮은 벽",
  sections: [
    { id: "high", label: "높은 구간", roofLabel: "높은 지붕", wallLabel: "높은 벽", width: 5078.6, height: 4170 },
    { id: "low",  label: "낮은 구간", roofLabel: "낮은 지붕", wallLabel: "낮은 벽", width: 1974.4, height: 2766 }
  ]
}
```

- `startSide`는 정면도에서 시작 외벽을 그릴 쪽이며 `"left"` 또는 `"right"`입니다. 좌우 미러링만 바꾸므로 수량에는 영향이 없습니다.
- `id`는 구간 안에서 유일해야 합니다. `width`, `height`, `L`은 모두 0보다 큰 유한 숫자여야 합니다.
- 지원 범위와 1·2·3단 예제, 케이스 수정 순서는 [SHAPE_EDITING.md](SHAPE_EDITING.md)를 따릅니다.
- 길이 방향으로 단면이 변하는 형상, 사선·곡면, 구멍·안뜰, 돌출·오버행 및 바닥 덮기는 현재 지원하지 않습니다.

## 2. 고정 전제

1. **패널 크기는 고정값** — 1800 × 600mm, 1장 면적 1.08㎡. 사용자가 수정하지 않음.
2. **바닥면은 덮지 않음** — 바닥으로 내려가는 잔여분은 잘라내는 것으로 간주. 계산 대상은 시작 외벽, 각 지붕, 단차 수직면, 반대편 외벽, 양끝 단면뿐.
3. **시작 외벽 위 기준 양방향 전개** — `shape.sections[0]` 쪽 지붕 모서리에서 시작해 한쪽은 시작 외벽을 바닥까지 직선 하강하고, 반대쪽은 모든 지붕·단차를 차례로 지나 마지막 외벽 바닥까지 접어 내려가는 것으로 계산. 현재 A/B/C에서는 시작 외벽이 높은 벽이므로 기존 시공 경로와 같습니다.
   - 1800mm 방향을 각 하강 전개 방향, 600mm 방향을 길이 방향 반복 배치로 사용.
   - 정면 ㄴ자 모서리와 지붕 단차는 절단선이 아니라 접힘선으로 취급.
   - 양방향 각각의 바닥 쪽 잔여띠는 스트립 재단과 1800×600 가로 밴드를 비교해 장수가 적은 쪽을 적용.
4. **가로/세로를 한 방향으로 고정하지 않음** — 다음을 자동 비교해 장수가 적은 쪽을 선택.
   - 긴 외피 바닥 잔여띠: 600폭 절단 스트립 vs 1800×600 가로 배치 → 장수가 적은 쪽(동률이면 이음매가 적은 가로 배치 우선 표기).
5. **케이스별 선택** — `CASES` 배열에 등록된 모든 케이스를 자동으로 나열하고 전환하며 확인. 현재 기본 데이터는 A/B/C 세 개.

## 5. 계산 로직

### 5.1 기본 파생값 — `base(c)`

`normalizeCaseShape(c)`가 먼저 단면을 정규화합니다.

```
totalW       = sum(section.width)
endArea      = sum(section.width × section.height)
roofPath     = sum(각 지붕 폭) + sum(인접 구간 높이차)
startRunW    = sections[0].height
profileRunW  = roofPath + sections[last].height
shellW       = startRunW + profileRunW
lenCols600  = ceil(L ÷ 600)           // 길이 방향 600폭 열 수
lenCols1800 = ceil(L ÷ 1800)          // 길이 방향 1800폭 열 수
trim600  = 600  − (L − 600×(lenCols600−1))   // 마지막 열 절단 자투리
trim1800 = 1800 − (L − 1800×(lenCols1800−1))
```

기존 A/B/C처럼 구간이 두 개인 경우 `roofPath = highW + abs(highH−lowH) + lowW`가 되어 이전 `drapeW`와 같습니다. `highW`, `lowW`, `highH`, `lowH` 등의 별칭은 이전 두 단 전용 화면과의 이행 호환용이며, 새 계산·시각화 코드는 `d.shape`의 파생값을 사용합니다.

### 5.2 긴 외피 전개 — `shellPlan(d)`
시작 외벽 위 모서리를 기준으로 두 방향의 `shape.runs`를 따로 계산합니다.
```
startRunW   = 시작 외벽 높이
profileRunW = 각 지붕 폭 + 각 단차 높이 + 마지막 외벽 높이

각 방향별:
fullRows   = floor(runW ÷ 1800)
mainSheets = fullRows × lenCols600
rem        = runW − fullRows×1800
```

### 5.3 바닥 잔여띠 — `bandPlan(rem, d)`
```
// 잔여 높이 처리: 두 방식 비교 후 적은 쪽 채택
stripsPerSheet   = max(1, floor(1800 ÷ rem))       // 패널 1장에서 뽑는 스트립 줄 수
stripSheets      = ceil(lenCols600 ÷ stripsPerSheet)
horizontalRows   = ceil(rem ÷ 600)
horizontalSheets = horizontalRows × lenCols1800
bandSheets        = min(stripSheets, horizontalSheets)      // 동률이면 가로 우선 표기

긴 외피 장수 = mainSheets + bandSheets
```

3D 비교에서는 두 run을 각각 독립 선택합니다(`runScenarioPlan`).

- 세로 본판: 길이축 600, 전개축 1800
- 가로 본판: 길이축 1800, 전개축 600
- 본판 뒤 잔여 폭: 자동 최소 / 가로 밴드 / 세로 스트립 중 선택
- 모든 선택안은 본판 뒤 미덮임, 길이 마지막 판 절단, 밴드 절단 잔여와 최종 부족(0mm)을 함께 반환

### 5.4 양끝 계단형 단면

- 구간 사이 경계는 별도 면으로 세지 않고, 외곽 계단형 단면을 **하나의 연속 면**으로 취급.
- `shape.sections`의 각 직사각형과 교차하는 격자 칸을 `Set`으로 합쳐 같은 칸을 한 번만 셉니다.
- 가로 고정안은 1800×600, 세로 고정안은 600×1800 격자를 단면 원점에서 연속 적용합니다.
- 각 고정 격자는 폭 방향의 실제 끝판 정보도 함께 계산합니다. `exact = totalW ÷ panelWidth`, `cols = ceil(exact)`, `lastUsed = totalW − panelWidth×(cols−1)`, `offcut = cols×panelWidth − totalW`. 이는 **한 줄의 폭 방향 계산**이며, 계단 단면 전체의 `장/면` 수량과 혼동하지 않습니다.
- `reviewedEnd.geometrySignature`가 현재 형상 서명과 일치하면 검토된 혼합 수량을 사용하고, 없거나 불일치하면 두 고정 격자 중 작은 값을 사용합니다: `endPerFace × 2면`.
- 면적 하한 검산식: `ceil(sum(section.width×section.height) ÷ 1,000,000 ÷ 1.08)`.

기존 A/B/C 두 단 형상에서의 비교식은 다음과 같으며, 일반화된 점유 격자 계산과 같은 결과를 냅니다.

- 가로 연속 격자: `ceil(lowH÷600)×ceil(totalW÷1800) + (ceil(highH÷600)−ceil(lowH÷600))×ceil(highW÷1800)`
- 세로 연속 격자: `ceil(highW÷600)×ceil(highH÷1800) + (ceil(totalW÷600)−ceil(highW÷600))×ceil(lowH÷1800)`

### 5.5 총합 · 손실률 · 발주
```
totalSheets = shellSheets + endSheets
totalPanel  = totalSheets × 1.08                          // ㎡
totalActual = L×shellW ÷ 1,000,000 + endActual
lossRate    = (totalPanel − totalActual) ÷ totalActual × 100
발주 수량    = ceil(totalSheets × (1 + 여유율))            // 여유율 0/3/5/7/10% 선택
```

## 7. 화면 구성 (섹션 맵)

1. **케이스·수량** — 가벼운 현재 케이스 요약과 같은 폭의 필요 수량·발주 권장 패널. 가로·세로 배치 비교 결과와 케이스 치수 상세를 함께 표시.
2. **덮는 방법** — 수치 나열 없이, 외피 연결·접힘·큰 면 우선·가로/세로 비교·양끝면 연속 배치를 짧게 설명.
3. **시각화** — **2D(천장 전개 / 양끝면 전개)**와 **3D(천장 전개 / 양끝면 전개)**를 명확히 분류.
   - 2D 천장·양끝면: 실제 비례 전개도와 가로/세로/혼합 배치 비교를 표시합니다. 각 도면 위에 마우스를 올리면 해당 배치의 투입 면적, 실제 피복 면적, 남는 면적을 표시합니다.
   - 3D 천장: 계단형 단면을 길이 방향으로 압출한 긴 외피만 캔버스 입체로 표시합니다. 드래그 회전 · 휠 확대/축소 · 더블클릭 시점 초기화, 별도 ▶ 순서대로 재생 버튼, 방향·잔여띠 비교를 제공합니다.
   - 3D 양끝면: 앞·뒤 계단 단면을 회전 가능한 축소 깊이 3D로 표시하고, 가로·세로·혼합 배치 버튼을 제공합니다. 전체 폭의 소수 장수, 설치 열 수, 마지막 판 사용폭과 절단 자투리는 구조물 밖에서 표시합니다.
   - 두 3D 메뉴는 마우스 hover로 선택 배치의 남는 면적을 표시하며, 구조물 내부에는 치수선을 놓지 않습니다. 양끝면은 케이스별 `shape.startSide`(A: 왼쪽, B·C: 오른쪽)에 맞춰 렌더링합니다.
4. **구역별 상세** — 구역 카드와 계획표(클릭 시 2D 시각화로 이동).
5. **비교** — `<details>`로 접힘(기본 접힌 상태). 면별 가로·세로·혼합 비교표 + 등록된 전체 케이스 선택 기준 비교표.
6. **검산** — `<details>`로 접힘(기본 접힌 상태). 6개 블록의 공식·중간값 전체 노출.

## 8. 알려진 제약

- 양끝 단면 혼합 최소값은 실측 재단 시뮬레이션이 아닌 케이스별 검토값이며, 현재 혼합 그림은 방향 개념도이지 개별 절단 조각의 1:1 재단도가 아님. 새 케이스는 검토값 없이도 가로·세로 고정 격자 최소안으로 계산되지만, 혼합 재단값을 총계에 쓰려면 실측/재단 검토 후 `reviewedEnd`를 등록해야 함.
- `reviewedEnd.geometrySignature`는 단면 치수와 반드시 일치해야 함. 불일치 시 검토값이 자동으로 비활성화되는 것이 정상이며, 새 형상 검토 전 임의로 서명만 갱신하면 안 됨.
- 겹침·이음 여유(시공 여유)는 수량에 포함되지 않음. 발주 여유율(0~10%)로만 보정.
- `shape.startSide`는 시각화 미러링에만 쓰이며 수량 계산에는 영향 없음(좌우 대칭이므로).
- 지원 형상 경계와 수정·검증 절차는 [SHAPE_EDITING.md](SHAPE_EDITING.md)를 기준으로 함.
